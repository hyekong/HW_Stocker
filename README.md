# 조립 PC 재고 관리 시스템 (HW_Stocker)

C언어와 연결 리스트(Linked List)를 활용하여 개발한 **실무형 조립 PC 부품 및 재고 관리 시스템**입니다. 
초기 1,000줄 이상의 단일 파일(Monolithic) 구조에서 벗어나, 기능별로 소스 코드를 분리(Modularization)하고 완벽한 예외 처리를 적용하여 안정성을 극대화했습니다.

## 주요 기능 (Key Features)

- **품목 및 재고 관리:** 부품(CPU, GPU, RAM 등 19개 카테고리)의 입고, 정보 수정, 삭제, 현황 조회
- **커스텀 세트(PC 조립) 관리:** 여러 부품을 묶어 하나의 '조립 PC 세트'로 구성하고 일괄 입/출고 처리
- **출고 및 장부 기록:** 단품 및 세트 출고 시 실시간 재고 차감 및 순이익 계산, `sales.txt`에 판매 이력 자동 기록
- **데이터 영구 저장:** 프로그램 시작/종료 시 `inventory.txt` 및 `custom_sets.txt`와 자동으로 동기화하여 데이터 보존

## 기술적 특징 및 트러블슈팅 (Technical Highlights)

본 프로젝트는 단순한 기능 구현을 넘어, **'사용자의 예측 불가능한 입력으로부터 프로그램을 어떻게 방어할 것인가'**에 집중했습니다.

1. **전면적인 예외 처리 (Defensive Programming)**
   - `while(1)` 루프와 입력 버퍼 클리어(`clearBuffer`)를 활용해 문자열 입력, 음수 입력 등 잘못된 값 입력 시 프로그램이 종료되거나 무한 루프에 빠지는 현상(버퍼 오버플로우)을 완벽 차단.
2. **소스 코드 모듈화 (Refactoring)**
   - 유지보수성을 높이기 위해 로직을 5개의 `.c` 파일과 1개의 `.h` 헤더 파일로 캡슐화 및 분리.
3. **안전한 파일 I/O 시스템 구축**
   - 띄어쓰기가 포함된 부품명(예: `Core i9-14900K`)을 저장하고 불러올 때 발생하는 파싱 오류를 해결하기 위해, 데이터 구분자를 공백에서 파이프(`|`) 기호로 변경하여 파일 손상 원천 차단.
4. **연결 리스트(Linked List)의 안전한 메모리 관리**
   - `malloc` 실패 시 방어 로직 추가 및 노드 삭제 시 정확한 `free()` 호출로 메모리 누수(Memory Leak) 방지.

## 프로젝트 구조 (Project Structure)

```text
📦 HW_Stocker
 ┣ 📜 main.c                 # 메인 메뉴 진입점 및 프로그램 루프
 ┣ 📜 inventory.h            # 전역 구조체(Product, CustomSet) 및 함수 프로토타입 선언
 ┣ 📜 inventory_add.c        # [F01] 신규 부품 등록 및 기존 부품/세트 입고 로직
 ┣ 📜 inventory_manage.c     # [F02, F03] 품목 정보 수정, 삭제 및 커스텀 세트 구성 로직
 ┣ 📜 inventory_business.c   # [F04, F06] 단품/세트 출고 처리 및 매출 장부 기록 로직
 ┗ 📜 inventory_system.c     # [F05, F07] 재고 리스트 출력, 파일 I/O 및 검색 유틸리티
```

## 컴파일 및 실행 방법 (How to Run)
```bash
# 전체 C 파일 컴파일
gcc -o hw_stocker *.c

# 프로그램 실행
./hw_stocker
```
